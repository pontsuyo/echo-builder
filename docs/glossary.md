# 用語集 - Echo Builders

## 1. 概要

本ドキュメントは、Echo Builders の実装前提として使用する用語をゲーム開発側の文脈に合わせて整理したものです。

## 2. コア用語

| 用語 | 定義 |
|---|---|
| **Echo Builders** | 音声指示で進行する 2D 建築ゲーム。プレイヤーの指示を NPC（子ども）が受け取り、建築タスクを実行する。 |
| **Player** | プレイヤー。実体はユーザー入力（音声）を開始・停止する主体。|
| **子ども（NPC）** | 建築タスクを受け取り、`game-commands.js` の解釈結果を基に `game-engine.js` が更新する受動的エージェント。 |
| **建築部位** | 家を構成する要素（屋根、壁、窓など）。本文献の「parts」または「partType」。|
| **ボイスコマンド** | 音声で入力された指示。最終的にテキスト化され、NPC への割当と実行可否判定に使われる。 |
| **曖昧理解** | 指示と NPC の解釈がずれる性質をゲーム体験として許容する設計方針。 |
| **建築進捗** | 各部位の `built` フラグがどれだけ完了したかを示す状態。 |

## 3. 音声関連の用語

| 用語 | 定義 |
|---|---|
| **Mistral Voxtral** | 音声認識（STT）を担う外部 API。 |
| **STT** | Speech-to-Text。Voxtral を使って音声を文字起こしする処理。 |
| **ask-voxtral-mic ボタン** | 音声入力を開始する UI 操作。 |
| **M キー** | キーボードから同等の音声入力開始を行う操作。
（移動キーは本仕様では未使用） |
| **文字起こし結果** | Voxtral から返却される確定テキスト。`game-commands.js` の入力候補に渡される。 |
| **プロキシ** | `mistral_proxy.py` が担う中継。API キーを直書きしない形で Voxtral 呼び出しを受ける。 |

## 4. ゲーム状態・処理状態

| 用語 | 定義 |
|---|---|
| **commandSession** | 1 回の音声認識で生じたコマンドの処理単位。`active`・`queue`・`cursor` を持つ。 |
| **commandSession.active** | 現在の音声入力セッションが有効かどうか。 |
| **commandSession.queue** | 順番待ちコマンド配列。NPC 割当前の FIFO 的な処理対象。 |
| **commandSession.cursor** | 次に処理する `queue` 要素のインデックス。 |
| **commandResultRows** | 最新の処理結果を画面表示するためのログ行。 |
| **Player listening** | `setPlayerListening(boolean)` 系で管理される「プレイヤーが音声入力を受け付ける状態」。|
| **ゲーム全体の進行状態** | 現時点では「入力受理」「コマンド解釈・配布」「建築更新」を中心に扱う。 |
| **ゲームループ** | `game-engine.js` が `requestAnimationFrame` 前提で画面更新を維持する枠組み。 |

## 5. 子ども（NPC）状態

| 用語 | 定義 |
|---|---|
| **state: walk / idle** | NPC の基本行動状態。`walk` は移動寄り、`idle` は待機寄りの状態を表現。 |
| **commandState: queued** | NPC がコマンドを待っている状態。 |
| **commandState: returnHome** | NPC が拠点へ戻る制御状態。 |
| **commandState: completed** | 現在割当の処理が完了した状態。 |
| **commandState: roam** | NPC の自由移動/待機関連の状態。 |
| **isListeningToPlayer** | 子どもが現在「指示を聞いている状態」かどうか。
ゲーム側が音声入力文脈を監視・切り替える基準となる。 |
| **preferredPartType** | 該当 NPC が優先的に扱いやすい部位タイプ。候補は今後拡張可能。 |
| **buildQuantity** | コマンドで指示された一度の建築対象個数。 |
| **lastHeardText** | 直近で聞いた原文（または要約）。 |
| **lastInterpretation** | 直近の解釈結果（曖昧理解適用後の命令）。 |

## 6. データ・コード構成用語

| 用語 | 定義 |
|---|---|
| **index.html** | ゲームの初期表示と UI コントロールの起点。 |
| **game-engine.js** | 状態更新・描画・初期化に関わるコアロジック。 |
| **game-commands.js** | 音声文字起こし結果を解釈して NPC へ割り当てるロジック。 |
| **game-data.js** | 部位定義、ルール、初期値などのデータ定義。 |
| **voxtral.js** | ボイス入力起点・STT 呼び出し系。 |
| **mistral_proxy.py** | ローカル代理 API。 |
| **R キー** | 全体状態を初期化してデモ実行しやすくするデバッグ操作。

## 7. 将来検討・今は未実装の用語

| 用語 | 定義 |
|---|---|
| **TTS** | 音声合成。現行実装の必須範囲外（将来検討）。 |
| **スコアリング** | 成績/数値評価の導線。現行実装では優先外（将来検討）。 |
| **理解度スコア** | 現在は仕様側の抽象用語であり、実装上の確定指標として未使用。 |
| **ワーカー幸福度** | 同上。現時点の実装仕様では使われていない。 |
