# プロダクト要求定義書 (PRD) - Echo Builders

## 1. 概要

**プロダクト名**: Echo Builders（仮称）
**コンセプト**: 音声指示で動く、曖昧理解を味方にする建築ゲーム

プレイヤーの音声入力を受けて、AIワーカーが建築行動を開始するゲームです。完全再現ではなく、読み違い・解釈のズレを通じて予期しない可愛さを体験することを狙います。

## 2. 目標

### 2.1 主要目標
- 音声入力から建築指示を開始できること
- ワーカーが指示を解釈して建築を進めること
- 指示内容と結果のズレが「わかる楽しさ」として再現されること
- 短時間で遊べる導線を維持すること
- 開始時（`R` リセット含む）に3パターン以上のゴール候補から1件をランダム選定し、目標構造を表示すること
- 建築完了時に「数・色・位置」を含む一致度スコアを確定し、だいたい合っている場合も高得点を取りやすくすること
- 余計な部品追加・破壊コマンド（未実装）を点数反映できる拡張仕様を持つこと

### 2.2 成功基準（記述方針）
- 実装が安定して起動し、音声入力→建築表示まで進むこと
- 指示が認識不能な場合のエラー表示と再試行導線があること
- 主要ゲームフロー（開始／入力／建築進行）が一貫して動作すること
- ユーザー実測で「意図が少しずれる」ことを見て楽しめること
- 3パターン以上の目標が開始・リセット時に選ばれ、完全一致でなくても高スコア帯に入ること

## 3. ユーザーストーリー

### 3.1 主要ユーザーストーリー

| ID | ユーザーストーリー | 受け入れ条件 |
|----|------------------|--------------|
| US-01 | 音声で建築指示を出せる | - マイク入力が有効化できる<br>- 音声がテキスト化される<br>- テキスト表示が見える |
| US-02 | AIワーカーが建築を開始する | - 音声指示完了後にワーカーの作業が開始する<br>- 建築の進行が画面で見える |
| US-03 | 読み違いを見られる | - 完成物が指示と完全一致しないことがあることを許容できる<br>- どこが異なったかを可視化できる |
| US-04 | ゲームを初期化できる | - リセットで初期状態へ戻せる<br>- 直近建築物を一度リセットできる |
| US-05 | 建築ゴールを確認できる | - 開始時にゴール候補をランダム選定できる<br>- ゴール情報を画面上で確認できる |
| US-06 | 結果の一致度を確認できる | - 建築完了時に得点が1回確定される<br>- 数・色・位置の一致率で「だいたい合っている」が高得点になる |

### 3.2 将来検討ユーザーストーリー

| ID | ユーザーストーリー | 受け入れ条件 |
|----|------------------|--------------|
| US-07 | 音声を再生して復唱を聞ける | - 声で結果を確認できる（音声合成） |
| US-08 | ワーカーを複数選択できる | - 各ワーカーに個性がある
- 選択時に表示が切り替わる |

## 4. 機能要件

### 4.1 コア機能

1. **音声認識システム**
   - リアルタイム入力受付
   - Mistral Voxtral API 経由の文字起こし
   - 転写結果の表示

2. **建築理解エンジン**
   - テキストから建築パラメータ抽出
   - ルール寄りの解釈（誤認識の発生余地あり）
   - 異常系時のフォールバック

3. **建築生成システム**
   - 屋根/壁/窓/ドアなどパーツを描画
   - 2Dピクセル表現
   - 進行状態に応じたアニメーション更新

4. **ゴール選定（p0）**
   - 起動時 (`startGame`) と `KeyR` リセット時にゴールを再抽選
   - `GoalPattern` は3件以上を前提
   - 画面上に目標構造を見やすく表示

5. **スコアリング（p0）**
   - 建築完了時に一致度スコアを1回確定
   - 数・色・位置の一致率で差分採点
   - 目標外の余計な部品を `-5` として減点
   - 破壊コマンドの減点 `-10` は次実装で有効化（現時点では破壊イベント未実装）

6. **UI/UX（最小）**
   - 音声入力コントロール
   - 転写文字列の表示
   - 進行状況表示
   - リセット

### 4.2 検討機能（現行は未実装）

1. **音声合成システム**
   - ワーカー復唱のTTS再生
   - ElevenLabs連携（将来方針）

2. **拡張モード**
   - ワーカー複数選択
   - ゲームモードの拡張

## 5. 技術要件

### 5.1 技術スタック

| カテゴリ | 技術/ツール |
|----------|-------------|
| フロントエンド | HTML5, CSS3, JavaScript (ES6+) |
| 描画 | Canvas API |
| 音声認識 | Mistral Voxtral API |
| ビルド | なし（単一HTML構成） |

### 5.2 API要件

1. **Mistral Voxtral API**
   - エンドポイント: `POST /v1/audio/transcriptions`
   - モデル: `voxtral-mini-latest` 系または環境差分により調整
   - ストリーミング/フォールバック双方を前提

2. **音声合成 API**
   - 現行実装では未使用（TTSは将来検討）

### 5.3 データ要件（暫定）

1. **建築パラメータ（暫定）**
```json
{
  "roofShape": "round|triangle|flat",
  "wallColor": "string",
  "windowCount": "number",
  "floors": "number",
  "decorations": "array"
}
```

2. **ワーカー属性（暫定）**
```json
{
  "id": "string",
  "name": "string",
  "voiceId": "string|null",
  "understandingBias": {
    "numbers": "0-1",
    "colors": "0-1",
    "shapes": "0-1",
    "abstract": "0-1"
  }
}
```

3. **ゴール/スコア仕様（暫定）**
```json
{
  "goalId": "goal-red-roof-3columns-door",
  "name": "赤い屋根・柱3本・中央ドア",
  "version": "1.0",
  "parts": [
    {
      "partType": "roof",
      "requiredCount": { "min": 1, "max": 1 },
      "weight": {
        "count": 10,
        "color": 10,
        "position": 0
      },
      "targetColorHex": "#d64545",
      "targetRoofShape": "triangle"
    },
    {
      "partType": "column",
      "requiredCount": { "min": 3, "max": 3 },
      "weight": { "count": 10, "color": 0, "position": 0 },
      "targetColorHex": "#f2f2ea"
    },
    {
      "partType": "door",

    "extraPart": -5,

```


## 6. 制約条件

1. **技術的制約**
   - ブラウザのマイクアクセス許可が必要
   - 文字起こし精度はMistral API依存
   - 音声合成（TTS）は現行で未実装
   - 破壊イベントのトラッキングは未実装（現行 `destroyedCount` は常時 0）

2. **設計制約**
   - 単一HTMLで起動する方針
   - 現時点ではオフライン音声機能は対象外

## 7. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 音声認識の精度不足 | ゲーム体験の悪化 | 再試行導線、簡略コマンドの優先対応 |
| API遅延 | 待ち時間の体感増加 | ローディングと進行表示 |
| 解析仕様のあいまいさ | 挙動が想定外になる | コマンド辞書と例外規則の整備 |
| 目標選定とスコア式の調整 | 初見の期待値がぶれる | しきい値・重みを公開し、最終確定時1回だけ計測 |
| 音声仕様の将来拡張 | 進行遅延 | 現行は最小要件に集中、将来機能を分離 |

## 10. 用語集

| 用語 | 定義 |
|------|------|
| Echo Workers | 音声指示を解釈して建築を進めるAIキャラクター |
| 曖昧理解 | 完全一致しない解釈を許容してゲーム体験を作ること |
| 復唱 | ワーカーが理解内容を報告する（現行では未実装） |
| スコアリング | 指示と結果の一致度を数値化する機能 |
| GoalPattern | 開始時に選択されるゴール仕様テンプレート |
| partial match score | 完全一致でなくても一定一致で高得点化される比率加点 |
