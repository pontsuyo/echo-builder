# プロダクト要求定義書 (PRD) - Echo Builders

## 1. 概要

**プロダクト名**: Echo Builders（仮称）
**コンセプト**: 音声指示で動く、曖昧理解かわいい建築ワーカーたちの世界

プレイヤーの音声指示を基に、AIワーカーが曖昧に理解しながら建築を進めるゲーム。完全な再現ではなく、「ちょっとズレる」「少し足りない」「思ってたのと違う」という余白と可愛さを楽しむ建築体験を提供します。

## 2. 目標

### 2.1 主要目標
- 音声入力による直感的なゲーム操作を実現
- AIの曖昧理解をゲームメカニズムとして楽しむ
- プレイヤーの創造性とAIの解釈のギャップから生まれるサプライズを提供
- 短時間でプレイ可能なカジュアルゲーム体験

### 2.2 成功指標
- プレイヤーが音声入力で建築指示を出せる
- AIワーカーが曖昧に理解した建築物を生成できる
- プレイヤーが「思ってたのと違うけど可愛い」と感じる体験を提供できる
- 1セッションあたり3-5分のプレイ時間
- 音声認識の成功率80%以上
- プレイヤー満足度調査で「楽しかった」回答が70%以上

## 3. ユーザーストーリー

### 3.1 主要ユーザーストーリー

| ID | ユーザーストーリー | 受け入れ条件 |
|----|------------------|--------------|
| US-01 | 音声で建築指示を出せる | - マイク入力が有効化できる<br>- 音声がリアルタイムでテキストに変換される<br>- 変換結果が画面に表示される |
| US-02 | AIワーカーが建築を開始する | - 音声指示が完了するとワーカーが動き出す<br>- 建築過程がアニメーションで表示される<br>- 建築完了までの時間が30秒以内 |
| US-03 | 曖昧理解の結果を見られる | - 完成した建物が画面に表示される<br>- プレイヤーの指示との違いが視覚的にわかる<br>- 「理解度スコア」が表示される |
| US-04 | ワーカーの復唱を聞ける | - 建築完了後、ワーカーが音声で復唱する<br>- 復唱内容が画面にテキスト表示される<br>- ElevenLabsの音声合成が使用される |
| US-05 | ゲームをリセットできる | - リセットボタンで初期状態に戻る<br>- 前回の建築物がクリアされる<br>- 新しいゲームセッションが開始できる |

### 3.2 追加ユーザーストーリー (将来的)

| ID | ユーザーストーリー | 受け入れ条件 |
|----|------------------|--------------|
| US-06 | 複数のワーカーを選択できる | - ワーカー選択画面が表示される<br>- 各ワーカーに個性がある<br>- 選択したワーカーの特性がゲームに反映される |
| US-07 | 建築物を保存・共有できる | - 完成した建物を画像として保存できる<br>- 共有用URLが生成される<br>- SNS共有ボタンが利用できる |
| US-08 | チャレンジモードをプレイできる | - 時間制限付きモードが選択できる<br>- スコアが表示される<br>- ランキングに登録できる |

## 4. 機能要件

### 4.1 コア機能

1. **音声認識システム**
   - リアルタイム音声入力
   - Mistral Voxtral APIとの統合
   - ストリーミング文字起こし
   - 中間結果の表示

2. **建築理解エンジン**
   - 音声テキストから建築パラメータ抽出
   - 曖昧理解モデル
   - 確信度計算
   - 誤解生成ロジック

3. **建築生成システム**
   - パラメータに基づく建物生成
   - 2Dピクセルアート描画
   - アニメーション表示
   - パーツ別描画（屋根、壁、窓、ドアなど）

4. **音声合成システム**
   - ElevenLabs APIとの統合
   - ワーカー個性に応じた声の選択
   - 復唱テキスト生成
   - 音声再生

5. **UI/UXシステム**
   - 音声入力コントロール
   - リアルタイム転写表示
   - 建築進捗表示
   - スコア表示
   - ゲームコントロール（リセット、再生など）

### 4.2 非機能要件

1. **パフォーマンス**
   - 音声認識のレスポンスタイム: 500ms以下
   - 建築生成時間: 2秒以下
   - フレームレート: 60fps維持

2. **互換性**
   - 対応ブラウザ: Chrome, Firefox, Safari, Edge
   - モバイル対応（タッチ操作）
   - デスクトップ対応（キーボード/マウス）

3. **セキュリティ**
   - APIキーの安全な管理（プロキシ経由）
   - ユーザーデータのローカル保存
   - プライバシーポリシーの表示

4. **アクセシビリティ**
   - キーボードナビゲーション
   - 画面リーダー対応
   - コントラスト比の確保
   - 音声フィードバックのボリューム調整

## 5. 技術要件

### 5.1 技術スタック

| カテゴリ | 技術/ツール |
|----------|-------------|
| フロントエンド | HTML5, CSS3, JavaScript (ES6+) |
| 描画 | Canvas API |
| 音声認識 | Mistral Voxtral API |
| 音声合成 | ElevenLabs API |
| ビルド | （なし - 単一HTMLファイル）|
| バージョン管理 | Git |

### 5.2 API要件

1. **Mistral Voxtral API**
   - エンドポイント: `/v1/audio/transcriptions`
   - モデル: `voxtral-mini-2602`
   - ストリーミング対応
   - エラーハンドリング

2. **ElevenLabs API**
   - エンドポイント: `/v1/text-to-speech`
   - 複数声のサポート
   - オーディオフォーマット: MP3

### 5.3 データ要件

1. **建築パラメータ**
   ```json
   {
     "roofShape": "round|triangle|flat",
     "wallColor": "string (hex)",
     "windowCount": "number",
     "floors": "number",
     "decorations": "array"
   }
   ```

2. **ワーカー属性**
   ```json
   {
     "id": "string",
     "name": "string",
     "voiceId": "string",
     "understandingBias": {
       "numbers": "0-1",
       "colors": "0-1",
       "shapes": "0-1",
       "abstract": "0-1"
     }
   }
   ```

## 6. 制約条件

1. **技術的制約**
   - ブラウザのマイクアクセス許可が必要
   - 音声認識の精度はMistral APIに依存
   - ElevenLabs APIの利用制限（レートリミット）

2. **設計制約**
   - 単一HTMLファイルで動作（インストール不要）
   - モバイルデバイスでのプレイを考慮
   - オフラインでは音声機能が利用できない

3. **法的制約**
   - API利用規約の遵守
   - プライバシーポリシーの表示
   - データ保護規制の遵守

## 7. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 音声認識の精度不足 | ゲーム体験の低下 | - 複数の理解候補を提示<br>- 確信度しきい値の調整<br>- 代替入力方法の提供 |
| APIの応答遅延 | ユーザーのイライラ | - ローディング表示<br>- 進捗インジケーター<br>- タイムアウト処理 |
| ブラウザの互換性問題 | 一部ユーザーがプレイできない | - ポリフィルの使用<br>- フォールバック機能<br>- 互換性テスト |
| ユーザーの期待とのギャップ | 滞在時間の短縮 | - チュートリアルの強化<br>- 期待値の調整<br>- フィードバック収集 |
